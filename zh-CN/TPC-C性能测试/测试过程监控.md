在YashanDB中，V$SYSTEM_EVENT/GV$SYSTEM_EVENT/V$SESSION_WAIT等视图显示了系统当前的等待事件信息，通过这些视图可以查看每个session正在等待的活动，并据此分析数据库的性能瓶颈点。

在TPC-C测试过程中，应关注等待事件的状态，并根据对应等待事件进行调优。

- log file sync

    描述：用户会话提交事务时，等待LGWR进程将REDO日志从日志缓存写入磁盘并返回确认的等待事件。

    解决方法：

    - 调高REDO_BUFFER_SIZE缓存的大小，提高REDO刷盘效率和回放效率。
    - 调大REDO_BUFFER_PARTS缓存的分区个数，某个会话产生的REDO日志被哈希分配到唯一的分区，设置合理的分区个数能减少并发冲突。
    - 如果事务负载主要是高并发低负载且REDO日志所在磁盘IO压力较大时，考虑调整COMMIT_LOGGING为BATCH模式。

- free buffer wait

    描述：访问数据块时触发BUFFER淘汰时的等待事件。

    解决方法：
    
    - 调大数据缓冲区配置参数DATA_BUFFER_SIZE的大小。
    - 增加DBWR_COUNT数据写磁盘线程的数量。

- buffer busy wait

    描述：当会话尝试访问缓存BUFFER中的某个数据块时，若该块已被其他会话锁定，当前会话需等待其他会话提交事务并解锁，即热点块访问。

    解决方法：
    
    - 调大数据缓冲区配置参数DATA_BUFFER_SIZE的大小。
    - 调整对象的PCTFREE参数，减少单块行数。
    - 使用分区表。

- db file scattered read

    描述：数据块物理读等待事件。

    解决方法：确认磁盘IO性能表现，如性能表现不佳建议使用更好的磁盘。

- extending data file

    描述：数据文件扩展等待事件。

    解决方法：预先创建更大的数据文件，避免出现文件扩展导致的性能损耗。
