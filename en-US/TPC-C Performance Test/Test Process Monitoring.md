In YashanDB, views such as V$SYSTEM_EVENT/GV$SYSTEM_EVENT/V$SESSION_WAIT display information on the current waiting events of the system. These views allow you to identify the activities that each session is waiting on, which assists in analyzing database performance bottlenecks.

During TPC-C testing, it is essential to monitor the status of wait events and perform optimization based on the corresponding wait events.

- log file sync

    Description: This wait event occurs when a user session commits a transaction and waits for the LGWR process to flush REDO logs from the log buffer to disk and return confirmation.  

    Solutions:  

    - Increase the size of the REDO_BUFFER_SIZE cache to improve REDO flush efficiency and replay performance.  
    - Increase the number of partitions in the REDO_BUFFER_PARTS cache. REDO logs generated by a session are hashed and assigned to a unique partition, and setting an appropriate number of partitions can reduce concurrency conflicts.  
    - If the transaction workload is primarily high-concurrency and low-load, and the disk IO pressure for REDO logs is high, consider adjusting COMMIT_LOGGING to BATCH mode.

- free buffer wait

    Description: This wait event occurs when accessing data blocks triggers buffer eviction.  

    Solutions: 
    
    - Increase the size of the data buffer configuration parameter DATA_BUFFER_SIZE.  
    - Increase the number of DBWR_COUNT threads responsible for writing data to disk.

- buffer busy wait

    Description: This wait event occurs when a session attempts to access a data block in the buffer cache, but the block is locked by another session. The current session must wait for the other session to commit the transaction and release the lock, typically due to hotspot block access.  

    Solutions: 
    
    - Increase the size of the data buffer configuration parameter DATA_BUFFER_SIZE.  
    - Adjust the PCTFREE parameter of the object to reduce the number of rows per block.  
    - Use partitioned tables.

- db file scattered read

    Description: This wait event occurs during physical reads of data blocks.  

    Solutions: Verify disk IO performance. If performance is poor, consider using higher-performance disks.

- extending data file

    Description: This wait event occurs when a data file is being extended.  

    Solutions: Pre-create larger data files to avoid performance degradation caused by file extensions.
